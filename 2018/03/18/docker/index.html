Docker常用命令

发表于 2018-03-18 | 分类于 Docker | 本文总阅读量

前言

很早之前部署项目的时候用到过，不经意间发现自己忘光了相关操作，幸好每次学习都做了笔记，当时本来就想发篇记录博客的，但本想着把深入的构建镜像以及搭建自己一套镜像服务都做好再写的，可惜时间不等人，忙忙就忘了，今天有时间还是记录下常用操作吧，不然某天我电脑瘫痪就GG了。

Docker

官网：www.docker.com
仓库:

docker官网仓库： hub.docker.com
阿里镜像仓库：dev.aliyun.com
163仓库(需注册):c.163yun.com/hub

安装docker

MacOS: 官网教程地址
Ubuntu: 官网教程地址
Ubuntu简单旧版安装：

uname -r //内核版本需要 > 3.10
sudo apt-get install -y docker.io //安装系统自带的docker(可能不是最新版本)
docker version //出现客户端和服务端版本

Docker初体验

docker pull [OPTIONS] NAME[:TAG]

拉取镜像
NAME必须指定，TAG表示版本，默认不写为latest最新版本
举例: docker pull hello-world
举例: docker pull tomcat
举例: docker pull nginx
举例: docker pull mysql
默认pull的地址为docker官方镜像地址:https://hub.docker.com/explore/
当然国内的也可以用163蜂巢的镜像地址:https://c.163.com/hub#/m/home/
libary开头的为163从docker官网复制过来的镜像
举例: docker pull hub.c.163.com/library/tomcat:latest
举例: docker pull hub.c.163.com/library/nginx:latest
举例: docker pull hub.c.163.com/library/mysql:latest

docker images [OPTIONS] [REPOSITORY[:TAG]]

查看本机镜像列表
举例: docker images

docker run [OPTIONS] IMAGE[:TAG] [COMMAND] [ARG]

运行docker容器
举例: docker run hello-world //如果没有此镜像则会自动从官方下载镜像
举例: docker run tomcat //通过命令行运行tomcat，按ctrl+c就可以结束
也可以带参数[OPTIONS]运行
举例: docker run -d hello-world //表示后台运行tomcat镜像，运行的镜像就成了一个容器
也可以带命令运行
举例: docker run learn/tutorial echo "hello word" //输出helloworld
举例: docker run learn/tutorial apt-get install -y ping //安装ping工具
举例: docker run lean/ping ping www.google.com //执行ping命令

docker ps

查看当前docker所运行的容器
举例: docker ps //列出所有运行着的容器
要查看所有容器，包括未运行的容器
举例: docker ps -a //显示所有容器

docker exec [OPTIONS] CONTAINER [COMMAND] [ARG]

在容器中运行命令
docker exec –help //查看帮助文档
-i,-t 参数最常用
举例：docker exec -it f4e5 bash //在某个容器里运行bash

docker stop CONTAINER_ID

停止某个容器(可以为容器的Id，也可以是容器的名称)
举例: docker stop 56f433965490

docker commit CONTAINER_ID IMAGE_NAME

将容器打包成一个新的镜像
举例: docker commit 56f tomcat/property //将56f容器打包成一个镜像，镜像名称自己起

常用命令

运行(Nginx)镜像

docker run -d -p 80:80 --name webserver nginx // 并将运行的容器起名为webserver

停止运行(Nginx的webserver)容器

docker stop webserver //–name指定的值
docker stop CONTAINER_ID //容器Id

删除所有容器

docker rm $(docker ps -a -q) //-q显示容器ID

删除镜像

docker rmi IMAGES_NAME, 如：docker rmi tomcat

在容器里运行bash命令

docker exec -it f4e5 bash //打开f4e5容器里的bash命令行界面

查看容器ip

docker inspect f4e5 | grep IPAddress

Docker运行操作

下载(Nginx)镜像

docker pull hub.c.163.com/library/nginx:latest
从163镜像中心下载nginx镜像
docker pull nginx
从官网镜像中心下载

运行Nginx

docker run hub.c.163.com/library/nginx //前台运行
docker run -d hub.c.163.com/library/nginx //后台运行

查看运行的容器里的内容

docker exec -it f4e5 bash
f4e5为运行的容器的id前几位数字
bash为运行的命令

Docker网络

docker run -d -p 8080:80 hub.c.163.com/library/nginx

将本机8080端口映射到容器的80端口，-p 代表开放端口
netstat -na | grep 8080 //查看本机端口

docker run -d -P hub.c.163.com/library/nginx

将本机随机的端口映射到容器的80和443端口

制作自己的镜像

Dockerfile
>

docker build [OPTIONS] [PATH]

PATH为执行的路径
docker build -t jpress:latest .
// -t 为生成的镜像配一个tag说明，jpress为名字，lastest为说明
docker build . //当前目录build

将war运行到Tomcat根目录(项目记录)

将本地8089映射到容器的8080端口

最后为镜像名称，如果为163蜂巢的镜像则应改为蜂巢的镜像名

-v为加载本地硬盘
-it为执行容器里命令开启
-i –interactive=false 打开STDIN，用于控制台交互
-t –tty=false 分配tty设备，该可以支持终端登录，默认为false

Docker私有仓库部署

两台机器: A:192.168.31.115(作为仓库) B:192.168.31.215(提交镜像者)

(1). A:下载仓库镜像

docker pull registry

(2). A:运行镜像(应该将本地某个目录映射到/tmp/registry中以保存镜像)

官方文档:点我
原来的1.0的数据保存在 /tmp/registry
2.0的数据存储在 /var/lib/registry
docker run -d -p 5000:5000 -v /opt/data/registry:/var/lib/registry registry

(3). B:先下载一个小镜像

docker pull busybox

(4). B:修改该镜像的tag

docker tag busybox 192.168.31.115:5000/busybox

(5). B:上传镜像

docker push 192.168.31.115:5000/busybox
出现问题不能https
官方配置:https://docs.docker.com/registry/insecure/
Mac上直接在docker图标的设置里添加Insecure-registry:192.168.31.115:5000,保存重启
Linux:/etc/docker/daemon.json

(6). B:检查是否上传成功

curl -XGET http://registry:5000/v2/_catalog
